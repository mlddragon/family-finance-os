# v0.4.0 Public Rehome Migration

This guide covers the move from private `Dillon_Finances` to public `family-finance-os`.

## GitHub

After the rehome PR stack merges, an owner action is required:

1. Rename the repository to `family-finance-os` in GitHub Settings.
2. Flip visibility to **Public** and complete the security settings in issue #66.
3. Archive `Family_Finance_planner` as a **private** research repository.

## Local clone

```bash
cd ~/GitHub
git remote set-url origin https://github.com/mlddragon/family-finance-os.git
# optional: mv Dillon_Finances family-finance-os
```

## Data roots

New default host paths:

| Mode | Path |
| --- | --- |
| Personal | `~/FamilyFinanceOS_Data` |
| QA synthetic | `~/FamilyFinanceOS_QA_Data` |

Legacy paths remain readable if you keep using them through Compose fallbacks, but Makefile defaults now target the new names.

Run:

```bash
python scripts/migrate_data_root.py
```

## Environment variables

| New | Legacy fallback |
| --- | --- |
| `FFOS_DATA_ROOT` | `DILLON_FINANCES_DATA_ROOT` |
| `FFOS_HOST_PORT` | `DILLON_FINANCES_HOST_PORT` |

## SQLite database

- New default file: `database/family_finance_os.sqlite3`
- Existing installs with `database/dillon_finances.sqlite3` continue to work without rename.

## Docker Compose project names

| Old | New |
| --- | --- |
| `dillon-personal` | `ffos-personal` |
| `dillon-qa` | `ffos-qa` |

Stop old containers before switching:

```bash
docker compose -p dillon-personal down --remove-orphans || true
docker compose -p dillon-qa down --remove-orphans || true
```
